APEXSHOT CAPTURE IMPLEMENTATION - QUICK REFERENCE
==================================================

CAPTURE QUALITY: 9/10 - HIGHLY PIXEL PERFECT

1. COLOR DEPTH & ACCURACY
   - X11: 24-bit RGB or 32-bit RGBA (8 bits per channel)
   - Wayland: 32-bit RGBA (8 bits per channel)
   - Format conversion: LOSSLESS (byte-level, no quantization)
   - Pixel formats supported: RGB24, RGB32, RGBA32, BGR24, BGR32, BGRA32

2. CAPTURE PIPELINE
   
   X11 Path:
   ├─ Visual detection (TrueColor, 24/32-bit)
   ├─ Byte order handling (LSB/MSB)
   ├─ GetImage request (ZPixmap, uncompressed)
   └─ Pixel format auto-detection
   
   Wayland Path (4-tier fallback):
   ├─ Tier 0: wlr-screencopy (50ms, direct protocol)
   ├─ Tier 1: grim subprocess (50ms, external binary)
   ├─ Tier 2: Portal screenshot (200-400ms, portal API)
   └─ Tier 3: ScreenCast + PipeWire (1-2s, video capture)

3. COMPRESSION & QUALITY LOSS
   - Capture stage: NO compression (raw bytes only)
   - PNG export: Lossless (PIXEL PERFECT) ✅
   - JPEG export: Lossy with configurable quality (1-100, default 85)
   - WebP export: Lossless by default
   - GIF recording: ~75% quality by default (configurable)

4. SCALING & HiDPI
   - rec_hidpi: DISABLED by default (no downscaling)
   - screenshot_retina_scale: DISABLED by default
   - Current behavior: Captures NATIVE resolution (correct)
   - @2x suffix: Optional filename annotation (advisory only)
   - Wayland blur surface: Uses 1/4 resolution (display-only, doesn't affect output)

5. OUTPUT FORMATS
   Format  | Extension | Compression | Color Depth | Alpha | Pixel Perfect?
   --------|-----------|-------------|-------------|-------|---------------
   PNG     | .png      | Lossless    | 8-bit RGB   | N/A   | YES ✅
   PNG     | .png      | Lossless    | 8-bit RGBA  | YES   | YES ✅ (with cursor)
   JPEG    | .jpg      | Lossy Q85   | 8-bit RGB   | NO    | ~99% ⚠️
   WebP    | .webp     | Lossless    | 8-bit RGBA  | YES   | YES ✅

6. KNOWN LIMITATIONS
   - Wayland cursor: NOT captured by wlr-screencopy (X11 only)
   - JPEG: Lossy format (not pixel-perfect)
   - X11 TrueColor requirement: Falls back on unsupported visuals
   - PipeWire timeout: 2-second hard limit
   - HiDPI downscaling: Feature exists but disabled (no data loss)
